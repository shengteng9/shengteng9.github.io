文件的分片

大文件由于自身体积较大造成上传时间较长，一旦中断重传必须重新开始，导致用户体验差开销的浪费。分片上传就有效的避免了以上问题。分片上传是将一个文件切割为一系列特定大小的数据片，分别将这些小数据片上传到服务端，全部上传完后再在服务端将这些数据片合并成为一个资源。

具体实现步骤

* 对文件进行md5加密，生成该文件的md5值。
* 后端判断该文件是否已经上传完成过，上传过则不再重传，前端进行上传结束展示。
* 如果未上传，对大文件分片。
* 逐片上传，上传完成后发上传结束请求。（如果是续传向后端取中断时文件分片下标，只上传未传部分。非续传则直接上传。)

自定义属性和Vue的指令

发表于 2019-05-03   |   分类于 Vue   |  

H5的自定义属性与Vue的指令形似，但二者有着根本上的差别，不能武断的说二者原理一样。

data-*全局属性是一类被称为自定义属性的属性，它赋予我们所有HTML元素上嵌入自定义数据属性的能力，并可以通过js与HTML之间进行数据的交换。所有这些自定义数据属性都可以通过所属元素的HTMLElement接口来访问。

Vue的实现则是拿到dom后通过其parse()函数Convert HTML string to AST.后解析出Vue的指令，根据其不同的作用做不同的操作。差别显而易见，前者可以通过所属元素的HTMLElement的接口访问，后者显然不行。范围不同,自定义属性只是js与HTML之间数据交换，后者的作用则要丰富的多。

React高阶组件&微信H5支付

发表于 2019-03-09   |   分类于 React   |  

1. 微信H5支付流程简述：

  1. 前端向后台发起支付
  2. 后台向微信支付发起请求
  3. 微信下单接口返回支付相关参数给后台，如支付跳转url（参数名“mweb_url”）
  4. 前端跳转到支付的url拉起微信支付。如需返回至指定页面，则可以在MWEB_URL后拼接上redirect_url参数，来指定回调页面
  5. 详细
中间问题

微信支付是完全跳离app，所以必须配置redirect_url跳回app。又由于尽管重定向到app,但不能获取支付状态。需要用户手动确认支付状态（是否完成支付），那么常规做法是重定向到支付发起页面，根据redirect_url传的参数去弹出对话框，让用户手动确认状态。

2. React高阶组件简介

  1. 高阶函数的概念：函数可以作为参数被传递。函数可以作为返回值被输出。
  2. 高阶组件（函数组件）：接受一个组件作为参数并返回一个新的组件的函数。高阶组件是一个函数并不是一个组件。
  3. 使用高阶组件：直接使用高阶组件名称并传入组件，或者使用ES6的装饰器。
  4. 高阶组件应用的两种模式：
  • 代理方式的高阶组件。返回的组件直接继承React.Component类，新组件扮演的角色是传入组件的一个代理，在新组件的render函数中，将被包裹组件渲染出来，除了高阶组件自己要做的工作，其余功能全部交给被包裹的组件。
  • 继承方式的高阶组件。采用继承的方式作为参数组件和返回组件，假如传入的组件是WrappedComponent,那么返回的组件是直接继承的WrappedComponent。

链表

链表存储有序的元素集合，但不同于数组，链表中的元素在内存中并不是连续放置的。每个 元素由一个存储元素本身的节点和一个指向下一个元素的引用(也称指针或链接)组成。下图展 示了一个链表的结构:

相对于传统的数组，链表的一个好处在于，添加或移除元素的时候不需要移动其他元素。然而，链表需要使用指针，因此实现链表时需要额外注意。数组的另一个细节是可以直接访问任何 位置的任何元素，而要想访问链表中间的一个元素，需要从起点(表头)开始迭代列表直到找到 所需的元素。
